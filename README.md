# ImageForge

ImageForge 是一个基于仓颉（Cangjie）与 ArkUI 开发的 HarmonyOS 原生 AI 图像创作应用。项目面向手机和平板端，提供内测登录、积分展示、图片编辑、文生图、历史记录和服务设置等功能，并通过 HTTP/SSE 与后端 PTP 服务通信，由服务端对接 ComfyUI 工作流完成图像处理。

当前版本重点是把移动端主流程跑通：用户输入访问码登录后，可以在首页查看功能入口，在创作页进行图片编辑或图片生成，并在历史记录中回看已完成任务。整体视觉采用浅色、玻璃拟态、蓝色强调的界面风格。

## 核心功能

### 登录与账户

- 内测访问码登录。
- 登录成功后同步用户名、订阅方案和剩余积分。
- 顶部悬浮栏展示 Beta 用户状态、积分和语言切换入口。

### 首页

- ImageForge 品牌展示。
- 四个核心能力入口：风格化修图、高清增强、背景重绘、文生图。
- 快速开始按钮可直接跳转到编辑或生成流程。

### 创作

创作页分为两个主模式：

- 编辑图片：支持风格化、高清增强、背景重绘三个工作流。
- 生成图片：输入文字描述后生成新图片，支持宽度、高度、步数、CFG 等参数。

编辑图片目前支持加载演示图片或粘贴图片 URL，再将图片字节上传到后端 `/api/edit`。生成图片使用 `/api/generate-stream` 接收服务端进度推送，并在完成后展示结果图片。

### 历史记录

- 本地保存已完成的编辑和生成任务摘要。
- 新任务会插入到历史记录顶部。
- 适合作为演示时的结果兜底与回看入口。

### 设置

- 服务连通性检查。
- 后端版本信息读取。
- 展示客户端与 Node 服务端、ComfyUI 的架构关系。
- 预留本地数据管理入口。

## 运行方式

1. 使用 DevEco Studio 打开项目根目录。
2. 等待 Hvigor 与 cjpm 依赖同步完成。
3. 连接 HarmonyOS 真机或启动模拟器。
4. 运行 `entry` 模块。

也可以在命令行中构建：

```bash
hvigorw assembleHap
```

## API 对接

客户端当前默认连接：

```text
https://ptp.matrixlabs.cn
```

主要接口包括：

- `POST /api/auth`：访问码登录。
- `POST /api/edit`：图片编辑，使用 multipart/form-data 上传图片。
- `POST /api/generate-stream`：文生图流式生成，返回进度和结果。
- `GET /api/health`：服务健康检查。
- `GET /api/version`：版本信息。

## 当前验收反馈与待优化事项

以下内容来自当前界面验收记录，作为下一轮 UI 与功能迭代的待办：

0. 登录界面可能得改成蓝色的，风格一致这一块。
1. 登录界面进入后，底部导航栏与界面底部间隔太大，遮盖内容太多，影响观感，能否改成和顶部导航栏一样的悬浮样式。
2. 首页部分，四个方框里面的字有点小，整体布局有点偏上。如果底部导航栏再往下移动的话，空出的空白部分会更多，可能得再塞点什么东西，或者把上面的已有内容再做大一点。
3. 创作界面编辑图片那一部分还是上传不了照片。可以继续研究是否接入系统相册选择、文件选择或更稳定的 URL/演示图加载方案。上面的“风格化”“高清增强”“背景重绘”和下面的“人像精修”等预设提示词交互逻辑不一致，一个是点了之后显示蓝色，一个是点击时有灰色阴影，或许改成一致会更好。编辑图片部分整体排版也需要重新梳理，颜色虽然好看，但是逻辑层次不够明显，可以按小模块加标题和分组。
4. 生成图片部分目前也看不到结果。按钮颜色不够好看，可以考虑改成和编辑图片部分“编辑图片 ~2 积分”按钮一样的蓝色渐变样式。
5. 历史记录还可以，当前效果基本能看。
6. 设置部分一直没动，后续需要继续完善。

## 后续迭代方向

- 统一登录页、主界面和创作页的蓝色视觉语言。
- 调整底部导航栏位置、遮挡范围和悬浮样式，让内容区域更舒展。
- 重做首页信息密度，放大核心入口或增加有价值的状态信息。
- 优先解决图片上传与结果图片展示问题。
- 统一创作页按钮、标签、预设项的选中和点击反馈。
- 将设置页从占位状态推进到可用状态，例如增加账户、清缓存、服务地址配置和历史清空能力。
